## `den.lib.__findFile`

The angle bracket resolver. See [Angle Brackets Syntax](/guides/angle-brackets/).

```nix
_module.args.__findFile = den.lib.__findFile;
```

## `den.lib.aspects`

Den aspects API. Provides aspect type definitions and internal resolution helpers.

### `den.lib.aspects.resolve class aspect`

**Internal.** Resolves an aspect for a given class (e.g., `"nixos"`), returning a
module with `imports`. Runs the full pipeline including entity instantiation.

### `den.lib.aspects.resolveImports class aspect`

**Internal.** Like `resolve` but skips entity instantiation (phase 4).
Use for nested resolution where only class imports are needed, such as
extracting `homeManager` modules from a host's aspect tree.

## `den.lib.resolveEntity name ctx`

<Aside title="Source" icon="github">
[`nix/lib/resolve-entity.nix`](https://github.com/denful/den/blob/main/nix/lib/resolve-entity.nix)
</Aside>

Resolves an entity by kind into an aspect-like record that feeds
[`den.lib.aspects.resolve`](#denlibaspectsresolve-class-aspect). `name` is the
entity kind (e.g. `"host"`, `"home"`, `"flake"`); `ctx` is the context attrset,
typically carrying the entity's config under its own kind key.

The returned record reads `den.schema.<name>` for `includes`, `excludes`, and
`collisionPolicy`, augments `ctx` with derived bindings for related entity kinds
(e.g. a `home`'s `host`/`user`) and any schema-level collision policies, and
installs those as `__scopeHandlers`. For aspect-bearing kinds it prepends a
parametric self-provide entry so the entity's root `.aspect` is resolved once its
scope is established. The record carries `name`, `meta`, `includes`, `excludes`,
`__entityKind`, and `__scopeHandlers`.

```nix
# Resolve an entity record, then run the pipeline for a class.
den.lib.aspects.resolve "flake" (den.lib.resolveEntity "flake" { });
```

This is the public resolver used by [`den.hosts`/`den.homes`](/reference/options/)
(`config.resolved`) and by the flake outputs.

## `den.lib.policy`

Policy effect constructors. See [den.policies reference](/reference/policies/)
for the full API.

### `den.lib.policy.resolve`, `.include`, `.exclude`, `.route`, `.provide`, `.instantiate`, `.spawn`, `.pipelineOnly`

Policy effect constructors used inside policy functions. See
[den.policies reference](/reference/policies/#policy-effects).

### `den.lib.policy.pipe`

Pipe builder API for quirk data routing. See
[den.quirks reference](/reference/quirks/#pipe-builder-api).

### `den.lib.policy.mkPolicy name fn`

Create a named policy record for use in `includes` lists.
See [Policies reference](/reference/policies/#denlibpolicymkpolicy-name-fn).

### `den.lib.policy.for entity policyOrList`

Conditional policy wrapper — fires only for a specific entity.
See [Policies](/explanation/policies/#conditional-policies).

### `den.lib.policy.when predicate policyOrList`

Conditional policy wrapper — fires only when predicate returns true.
See [Policies](/explanation/policies/#conditional-policies).

## `den.lib.policyInspect`

### `den.lib.policyInspect.inspect { kind, context }`

Lightweight policy inspection utility. Calls resolve functions directly
without running the full pipeline. Returns matching policies with their
targets and routing type.

See [den.policies reference](/reference/policies/#denlibpolicyinspectinspect)
for full documentation.

## `den.lib.nh`

<Aside title="Source" icon="github">
[`nix/lib/nh.nix`](https://github.com/denful/den/blob/main/nix/lib/nh.nix)
</Aside>

Generates [nh](https://github.com/viperML/nh) wrapper scripts for each
host and home in your fleet. Each wrapper is a shell application that
calls `nh os`, `nh darwin`, or `nh home` with the correct flake
attribute, so you can build/switch with `nix run .#<hostname>`.

### `den.lib.nh.denPackages args pkgs`

Returns an attrset of all host and home wrapper packages, suitable for
`perSystem.packages`:

```nix
perSystem = { pkgs, ... }: {
  packages = den.lib.nh.denPackages { fromFlake = true; } pkgs;
};
```

### `den.lib.nh.denShell args pkgs`

Returns a `mkShell` with `nh` and all wrapper scripts on `PATH`:

```nix
# Non-flake usage
options.den.sh = lib.mkOption {
  default = den.lib.nh.denShell {
    fromFlake = false;
    outPrefix = [ "flake" ];
  } pkgs;
};
```

### `den.lib.nh.denApps args pkgs`

Returns a list of all wrapper derivations (hosts + homes).

### `den.lib.nh.hostApps args pkgs`

Returns a list of host-only wrapper derivations.

### `den.lib.nh.homeApps args pkgs`

Returns a list of home-only wrapper derivations.

### Args

All functions take an `args` attrset:

| Field | Type | Default | Description |
|---|---|---|---|
| `fromFlake` | `bool` | `true` | Use flake ref (`path#attr`) or file ref (`--file path attr`) |
| `fromPath` | `str` | `"."` | Flake path or file path |
| `outPrefix` | `list` | `[]` | Prefix prepended to the output attribute path |
| `defaultAction` | `str` | `"build"` | The default action to perform if no args are provided |
| `defaultArgs` | `[str]` | `[]` | Additional default args to add to each invocation, e.g., `--print-build-logs` |

### Usage

After adding `denPackages` to your `perSystem.packages`:

```console
# Build igloo via nh (defaults to "build" action)
$ nix run .#igloo

# Switch igloo via nh
$ nix run .#igloo -- switch

# Any nh flags work
$ nix run .#igloo -- switch --dry
```
