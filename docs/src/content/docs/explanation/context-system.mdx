<Aside type="caution" title="Legacy API — Scheduled for Removal">
`den.ctx` is a compatibility shim. It forwards entries to `den.aspects` and
`den.schema.<kind>.includes` with deprecation warnings at evaluation time.
It will be removed after the first stable release.

**Migrate to:**
- `den.aspects` for aspect definitions
- `den.schema.<kind>.includes` for entity inclusion
- `den.policies` for entity topology and routing

See [Migrating from den.ctx](/guides/migrate-ctx/) for step-by-step instructions.
</Aside>

## What it does

`den.ctx` was the original API for declaring aspects and binding them to entity
resolution stages. It combined aspect definitions, cross-entity providers, and
pipeline transitions in a single namespace.

The compatibility shim in `modules/compat/ctx-shim.nix` accepts `den.ctx.*`
entries and forwards them:

```
den.ctx.<name> = { ... }
  ↓
den.schema.<name>.includes = [ <value with deprecation warning> ]
```

Each forwarded value emits a `lib.warn` at evaluation time telling you exactly
what to change.

## What replaced it

The `den.ctx` namespace was split into three independent concerns:
`den.aspects` for aspect definitions, `den.policies` for entity topology,
and `den.schema.<kind>.includes` for explicit activation.

For the full old→new mapping and step-by-step migration instructions, see
[Migrating from den.ctx](/guides/migrate-ctx/).
